# SEASATS测试艇活动监测设计

**日期：** 2026-07-01

## 目标

生成一个进入预置应用列表的海事告警指挥看板：`SEASATS测试艇活动监测`。应用结合客户提供的两份 Excel 附件，展示 SEASATS 测试艇目标清单、SEASATS 55 的 AIS 轨迹回放、重点区域低速/往返活动、疑似 AIS 中断和综合研判评分。

## 数据边界

第一版真实附件数据包含 79 艘目标清单和 1 艘完整轨迹。目标清单来自 `副本1a8083ce4a7ced5847024a560e3ed22b.xlsx`，字段包括 MMSI、船名、最新位置、速度、长宽；轨迹明细来自 `副本0cb4b68fa1a67179a0368da8eb82dff6.xlsx`，19091 条轨迹点均为 `mmsi=338414915`，可关联到 `SEASATS 55`。

运行时前端不解析 Excel。实现时通过项目脚本把附件归一化为 `src/data/seasatsPayload.json`。后续如需补全另外 78 艘轨迹，可以接入生成扩展轨迹，但内部必须保留 `轨迹来源标记`；界面不强制显眼标注模拟，但不得把可知的补全轨迹明确声称为真实 AIS 轨迹。

## 研判规则

- 船名命中：大小写不敏感匹配 `^SEASATS?\s*(TEST|数字)`。
- 尺寸命中：`4*2` 为强命中，`3*2` 作为近似命中并标记尺寸偏差。
- 速度归一化：清单文件 `速度（除10）` 先除以 10 得到节；轨迹文件 `speed` 已是节。
- 低速活动：归一化后 `0 <= speed <= 3`。
- 持续低速告警：同一目标在同一重点区域内累计或连续低速覆盖至少 10 分钟。
- 疑似往返活动：同一重点区域内，轨迹片段路径距离 / 首尾位移 `>= 3` 且持续至少 10 分钟。
- 疑似 AIS 中断告警：同一 MMSI 相邻轨迹点时间差 `> 30` 分钟触发，`> 6` 小时高风险；中断前后靠近重点区域时提升优先级。
- 综合评分：船名、尺寸、低速、重点区域、往返、AIS 中断共同评分，不要求所有条件同时满足。缺少轨迹的 78 艘保留为最新位置/待核验对象。

## 应用形态

首屏是工作型指挥看板，不做营销页。顶部显示应用名、数据窗口、目标数、告警数、当前回放时间和底图状态。左侧是目标列表和筛选器，支持按状态、区域、尺寸和轨迹来源筛选。中间使用 MapLibre GL + Esri World Imagery 在线卫星瓦片，叠加目标点、重点区域、SEASATS 55 轨迹片段、AIS 中断点和告警事件。右侧显示告警列表和选中目标详情，解释每条状态的源数据。底部提供时间轴和回放控制，用于切换轨迹片段和日期范围。

地图技术遵循 `.claude/skills/map-timeline-replay/SKILL.md`：MapLibre 单实例、GeoJSON 业务图层更新、不因选择或回放重建地图；Esri 公共瓦片不需要密钥；瓦片失败时显示 `底图加载受限`，但业务图层仍可见。

## 预置应用登记

新场景目录为 `scene/seasats-test-craft-monitor/`。`.factory/app.json` 使用：

- `slug`: `seasats-test-craft-monitor`
- `name`: `SEASATS测试艇活动监测`
- `type`: `command-dashboard`
- `source`: `preset`
- `entry`: `static-vite`

根目录 `.factory/scene-catalog.json` 将该 slug 登记为 `application` 表面，进入预置应用列表。

## 验证

实现必须包含纯逻辑测试，覆盖速度归一化、船名/尺寸命中、轨迹分段、区域命中、AIS 缺口、低速/往返告警和 MapLibre GeoJSON 适配。完成后运行新场景 `npm test` 和 `npm run build`。最后启动 Vite dev server，在浏览器中检查地图画布、目标点、重点区域、轨迹、告警列表和响应式布局。
