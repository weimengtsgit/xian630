# 航母舰载机归属推断工具地图渲染设计

**日期：** 2026-06-24

## 目标

将“全球起降热力地图”由内嵌 SVG 网格替换为可缩放、可拖拽的真实地图。地图采用
MapLibre GL 和公开 Esri World Imagery 卫星影像底图，完整展示起降事件与航母已知
轨迹，并在 Portal 容器及较小桌面视口中避免被裁切。

本次只改变地图渲染边界和相邻布局，不改变 ADS-B、航母位置、海陆掩膜、归属研判或
mock 数据输入边界。

## 现状与问题

现有组件以全局等距投影将数据绘制到 SVG 中。所有 mock 事件集中在东经 134 至
142 度，而页面同时强制最小尺寸 1366×760 并隐藏溢出。因此在 Portal 或 1280×720
桌面视口中，点位位于被截断的右边缘，只留下空白经纬网。

SVG 只有经纬线与简化的本州提示，未提供真实地理底图、缩放或平移能力，无法承担
“全球地图”这一用户界面承诺。

## 方案

### 地图边界

新增一个独立的 MapLibre 地图组件，替代 `HeatMap` 中的 SVG 绘制代码。组件接收现有
`events`、`carriers`、`selectedIcao`、`selectedCarrierId` 与选择回调，不读取或修改
归属判定状态。

组件内部将输入转换为以下 GeoJSON 数据源：

1. 海上疑似起降点：起飞、降落以不同红橙颜色表示；选中 ICAO 的点位放大并增加描边。
2. 审计点：陆地和未知点以不同颜色的空心符号表示，且不参与疑似舰载机判定。
3. 航母航迹：每艘航母一条蓝色轨迹；选中航母时增强线宽和不透明度。

点击起降点继续调用现有 `onSelectEvent`，使左侧飞机行展开；点击航迹继续调用
`onSelectCarrier`，使列表筛选与航迹高亮保持双向联动。地图初始化时，视野按可见事件
与轨迹的联合范围自动定位；用户仍可自由拖拽和缩放到全球任意区域。

### 底图和可用性

MapLibre 样式使用仓库已有场景的 Esri World Imagery 栅格瓦片地址，保留必需的
attribution。地图包含标准缩放与比例尺控件。

加载或请求底图瓦片失败时，不阻塞 GeoJSON 业务图层和现有控件；面板显示“底图加载
受限”状态，用户仍可看到起降点和航迹。该降级仅覆盖网络瓦片不可用，不替换或伪造
数据来源。

### 交互与回放

既有“海上起降、航母航迹、审计点”图层开关、悬浮信息与时间回放控制保留为 React
覆盖控件。时间回放改变可见事件和每条航母轨迹的 GeoJSON 数据，而不是重新创建地图
实例。初始视野使用全部数据；用户拖动回放条后，视野不自动跳动，避免干扰手动浏览。

### 响应式布局

移除应用根节点固定的 `min-width: 1366px`、`min-height: 760px` 以及由此导致的裁切。
主面板使用带 `minmax(0, …)` 的网格轨道，确保右侧面板可收缩。窄于桌面双栏阈值时，
改为纵向布局，地图维持可用的最小高度而不隐藏内容。地图容器在挂载后和尺寸变化时调用
`resize()`，保证 Portal iframe 尺寸变化后画布不会保持旧尺寸。

## 依赖与文件范围

- 在场景自身 `package.json` 增加与现有场景一致的 `maplibre-gl` 版本。
- 使用 MapLibre 官方 CSS；不引入 React 地图库或地图服务密钥。
- 将 GeoJSON 转换、时间窗口裁剪和联合范围计算提取为无副作用的逻辑模块，便于测试。
- 更新地图组件、相关样式与场景 README，说明公开在线卫星瓦片的运行依赖和底图受限状态。

## 验证

1. 为 GeoJSON 转换、事件/轨迹时间窗口裁剪、初始地图范围各写一个单元测试。
2. 构建场景并运行全部逻辑测试。
3. 在 1280×720 和 Portal 相近尺寸中验证 MapLibre 画布可见，海上点位、航母轨迹、图层
   开关、点位选择、时间回放均正常工作。
4. 在浏览器中确认没有控制台异常；瓦片请求不可用时，确认状态提示出现且业务图层仍在。

## 非目标

- 不接入真实 ADS-B、航母位置库或海陆 GIS 服务。
- 不改变归属判定阈值、事件识别和已有 mock 数据语义。
- 不实现离线瓦片缓存、航海图服务或地图服务账号管理。
