# Use a stable document layout inside generated application projects

Generated application projects will keep user-facing project documents under `docs/`, application code under the normal source tree, and Factory-owned metadata under `.factory/`. A `.factory/project-docs.json` index links each Markdown project document to the source machine execution contract and generation attribution, while `output.json` remains in the run/artifact audit system rather than becoming the default user-facing project file.

Project documents are generated by the factory after a step's machine execution contract has validated successfully. Document projection is idempotent by source contract checksum; a projection failure records a warning and leaves the task execution contract authoritative rather than silently fabricating or partially applying a document.

Application-project file APIs are rooted at the generated application's project directory only. They must reject absolute paths, `..` traversal, symlink escapes, hidden audit/run directories, dependency/build outputs, and over-limit files before reading content.
