# 生成任务步骤执行记录设计

## 目标

在软件工厂门户中，让操作员能够监控并审计一个生成任务的阶段执行，而不把模型隐藏推理当作产品日志展示。

> 注：当一次生成任务带有协作智能体参与计划（动态计划）时，任务区按**协作泳道**渲染参与智能体卡片；没有计划的旧任务仍回退到下文的固定六阶段卡片矩阵。两种形态共用同一套执行记录、抽屉、产物与 SSE 机制。

## 术语

- **步骤执行记录**：一个生成任务阶段的可审计记录，由系统状态日志、面向用户的分析工作日志或公开摘要、命令执行输出和产物组成。
- **高级审计**：步骤执行记录中默认折叠的输入、提示词、结构化模型输出与完整原文区域。

## 阶段模型

带动态协作计划的生成任务由 `collaboration.DefaultPlan` 种子出 12 个协作智能体（条件加入安全审查智能体时为 13 个），按 analysis / generation / delivery 三条泳道分组。没有计划的旧任务回退到固定六阶段卡片，顺序固定：

1. 需求分析
2. 方案设计
3. 代码生成
4. 测试验证
5. 镜像构建
6. 部署

镜像构建与部署在视觉上归为“构建与部署”，但必须是可独立失败、独立重试和独立审计的卡片。两种形态（动态泳道 / 固定六卡片）共用同一套执行记录、抽屉、产物与 SSE 机制。

## 任务区

- 当任务带有协作计划时，任务区按泳道（analysis / generation / delivery）渲染参与智能体卡片；否则回退为固定六卡片矩阵。两种形态都保持中间列高度稳定，不挤压需求澄清和对话输入区。
- 每张卡片显示阶段名称、对应智能体角色、状态、运行或完成耗时、尝试次数、最新关键记录和未读数。
- 使用真实状态和里程碑，不显示估算进度百分比。
- 卡片默认显示最新一次尝试；重试后的旧尝试不被覆盖。
- 新运行阶段不自动抢占当前查看内容，只在卡片上显示未读标记。

## 步骤执行记录抽屉

点击任一卡片后，从右侧打开抽屉。抽屉覆盖右侧智能体列表的一部分，不改变中间列布局。

抽屉包含固定页签：

1. **概览**：状态、耗时、尝试次数、真实里程碑、失败原因和可执行操作。
2. **执行记录**：按时间排序的系统状态日志、公开摘要和命令输出。
3. **产物与审计**：输出契约、测试报告、构建日志和其他 artifact；高级审计内容默认折叠。

运行中的执行记录初次打开时自动跟随最新内容。用户向上滚动后停止跟随，新内容以未读数提示；点击提示后回到末尾并恢复跟随。历史尝试和已完成步骤不自动滚动。

## 内容边界

### Claude 阶段

需求分析、方案设计和代码生成显示两类内容：

- 实时阶段活动，例如读取确认需求、加载生成能力包、参考场景蓝本、写入应用文件和校验 manifest。
- 模型明确输出的公开摘要与结构化结论。

不得展示隐藏 reasoning、思维链或未审计的内部推理字段。

### 命令阶段

测试验证、镜像构建和部署实时追加真实命令输出。完成后可查看完整 stdout 和 stderr。

## 恢复与保留

- 前端先获取持久化快照，再订阅 SSE 增量；刷新、重连或重新打开门户后必须恢复当前任务、卡片状态和步骤执行记录。
- 每个步骤尝试保留独立的执行记录与 artifact。
- 单个文本 artifact 最大 10 MiB；超过上限时保留最新内容并记录截断状态。
- MVP 不自动删除任务审计记录；服务级 JSONL 日志继续使用独立轮转。

## 操作规则

- 运行中的当前阶段可在抽屉中取消整个生成任务。
- 仅当前失败阶段的最新尝试可显示“重试当前阶段”。
- 已完成、排队中与历史尝试只读。

## 可见性与安全

MVP 假设门户运行在本地受信任操作员环境中。所有门户访问者可查看高级审计内容，但内容必须以纯文本渲染，不执行或内嵌 HTML。服务端继续限制 artifact 只能从 `.factory-runs` 读取，并在写入执行记录前脱敏环境变量、令牌和授权头。

## 非目标

- 不展示或复原模型隐藏推理。
- 不伪造步骤百分比进度。
- 不在本阶段加入登录、角色或跨用户的审计权限模型。

## 实现追溯

本节记录 Tasks 1-6 已落地的实现，映射到上文设计需求；不改变已批准的交互决策。

### 已实现记录种类（`model.ExecutionRecordKind`）

| 种类 | 对应设计需求 |
|---|---|
| `system` | 卡片与抽屉概览中的系统状态/里程碑（真实状态，非估算进度） |
| `activity` | Claude 阶段的实时安全工具活动（Read/Grep/Glob/Edit/Write → 脱敏相对路径） |
| `summary` | 模型明确输出的公开摘要与结构化结论 |
| `command_stdout` | 命令阶段（测试/构建/部署）实时 stdout |
| `command_stderr` | 命令阶段实时 stderr |
| `error` | 失败原因，用于抽屉概览与失败重试判断 |

### REST 端点

| 端点 | 设计需求 |
|---|---|
| `GET /api/jobs/:id/execution-summary` | 六张卡片的状态/尝试/最新记录水合 |
| `GET /api/jobs/:id/steps/:stepID/execution-records?attempt=N&before_sequence=N&limit=200` | 抽屉“执行记录”页签，按时间升序分页；步骤缺失或不属于该任务返回 404 |
| `GET /api/jobs/:id/artifacts` | 抽屉“产物与审计”页签 |
| `GET /api/artifacts/:id/content` | artifact 内容（纯文本渲染；仅 `.factory-runs` 根，路径穿越受保护） |

### SSE 事件

| 事件类型 | 设计需求 |
|---|---|
| `step.record.appended`（`data` 为一条已持久化记录；信封携带单调 `seq`） | “恢复与保留”——前端以 `seq` 做缺口检测，缺口时回到 REST 重水合；SSE 不携带文件内容或隐藏推理 |

### 安全与保留行为（已落地）

- 操作性 `input.json`/`prompt.md`/`output.json` 审计时**不被原地修改**；脱敏副本写入 `attempt-N/audit/` 并登记为 artifact。
- 单文本 artifact 上限 10 MiB，超额保留**最新**字节（UTF-8 安全），前缀 `[TRUNCATED: retained latest 10485760 bytes]`，并置截断标记。
- 脱敏匹配 `api_key`/`token`/`secret`/`password`/`authorization`（大小写不敏感，`key=value` 与 HTTP 头两种形式）→ `[REDACTED]`。
- Claude 阶段记录安全工具活动、公开 `workLog`，以及（方案 B，已放宽原"永不展示"约束）模型的 **`thinking`/推理过程**：解析真实 Claude Code CLI `stream-json`（`assistant.message.content[]` 中的 `thinking` 块）为 `thinking` 记录、`Write`/`Edit` 工具调用为 `file_delta` 记录（`新建/编辑 <path> +N -M`，实时展示代码生成进度）。`thinking` 内容仍经过执行记录持久化前的脱敏关卡（凭证 → `[REDACTED]`），并按 ≤4 KiB 分块。**仅在真实 CLI 模式下出现**（`FACTORY_FAKE_CLAUDE` 未设置）；`FakeClaudeRunner` 不产出这两类记录。
- 卡片默认显示最新尝试；旧尝试保留可查；取消仅限当前运行步骤（取消整个任务），重试仅限当前失败步骤的最新尝试。
- 刷新/重连/`seq` 缺口时，前端先取 REST 快照再应用 SSE 增量；详细执行记录按 step+attempt 分页，仅在抽屉打开时拉取。
