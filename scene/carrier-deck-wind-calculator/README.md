# 甲板风实时计算器

航母活动区域甲板风条件评估看板（command-dashboard 类预置应用）。

## 场景口径（客户原始）

- 接入全球气象预报系统公开格点风场数据。
- 筛选全球所有已知航母活动区域的 10 米高度风速和风向。
- 舰载机起降所需甲板风最小值为 **20 节**。
- 用风速叠加航母最大航速 **30 节**，计算可实现的甲板风范围。
- 推算每艘航母当前是否具备“无弹射器辅助”或“安全着舰”的风力条件。
- 每 **5 分钟更新**。

## 数据来源说明（重要）

- 本应用全部数据为 **mock / 演示数据**，结构按未来 GFS 类公开格点风场适配。
- 不接入真实气象 API、不使用任何密钥、不依赖后端或云服务。
- 数据源名称固定显示为 `公开格点风场（mock）`。
- 顶部“每 5 分钟更新”为客户口径；本地演示使用约 6 秒的 tick 推进“最近刷新时间”、风速与判定状态，以便直观看到状态变化。

## 计算口径

设海域真实 10 米风速为 `W`（节）：

- **可实现甲板风范围** = `[|W − 30|, W + 30]` 节（航母最大航速 30 节，通过航向自合成甲板风）。
- **“无弹射器辅助” / “安全着舰”** 两个条件均按同一依据分级——即可实现范围 `[lo, hi]` 与客户最小值 `20 节` 的关系：
  - 满足：`lo ≥ 20`（任意航向甲板风均达标）
  - 临界：`lo < 20 ≤ hi`（顶风航行可达 20 节）
  - 不满足：`hi < 20`（无法达到 20 节）

> **客户口径 / 演示分级 分离说明**：客户仅提供 `20 节` 最小值、`30 节` 航速、可实现范围 `[|W−30|, W+30]` 三项参数。看板中的 `满足 / 临界 / 不满足` 分级是基于上述客户参数派生的演示状态分层，并非客户提供的运营判定，实际部署以客户最终确认为准。`不满足` 为看板定义的状态，但以 `30 节` 航母为例 `hi = W + 30 ≥ 30 > 20`，正常条件下不会出现——已在图例中如实标注。

## 运行

```bash
npm install
npm run dev      # 本地开发，默认 http://127.0.0.1:5173
npm run build    # 构建到 dist/
npm run preview  # 预览构建产物
```

## 容器化

```bash
podman build -t software-factory/carrier-deck-wind-calculator:latest .
podman run --rm -p 18080:80 software-factory/carrier-deck-wind-calculator:latest
```
